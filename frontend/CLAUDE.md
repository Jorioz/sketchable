# Sketchable — Frontend

A mobile-first drawing app built with React 19, TypeScript, Vite, Tailwind CSS v4, and Fabric.js.

## Stack

| Layer | Library |
|---|---|
| Framework | React 19 + TypeScript (strict) |
| Bundler | Vite 8 |
| Styling | Tailwind CSS v4 (`@import "tailwindcss"` — no config file) |
| Canvas | Fabric.js v7 |
| UI components | shadcn (Base UI primitives) |
| Icons | react-icons — **Ionicons 5 pack only** (`react-icons/io5`) |
| Auth | Auth0 (`@auth0/auth0-react`) — Google sign-in |
| State | React context (`DrawingContext`, `SessionContext`) |

## Key constraints

- **Max width `max-w-screen-sm` (640 px)** — the canvas and all controls are constrained to this and centered.
- **Canvas is always square** (`aspect-square`). The Fabric.js layers are imperatively sized via `ResizeObserver`.
- **`SketchCanvas` owns all Fabric.js refs** — never reach into them from outside. Undo/redo handlers are registered into `DrawingContext` via `registerHandlers` so sub-components can trigger them without knowing about Fabric.

## Auth & pairing

- **Gate**: `App.tsx` renders `OnboardingScreen` (signed out) → `PairingScreen`
  (signed in, unpaired) → `MainApp` (paired). Everything below the gate can call
  `useSession()` for `userId`/`pairId`/`partnerId`/`code` — guaranteed present.
- **userId** comes from the server (`pairing.userId`), which the Lambda derives
  from the verified Auth0 `sub`. The client never asserts its own identity.
- **API auth**: `lib/api.ts` sends `Authorization: Bearer <token>`. `App.tsx`
  registers Auth0's `getAccessTokenSilently` via `setTokenProvider` once signed
  in. Tokens are JWTs only if `VITE_AUTH0_AUDIENCE` is set (required).
- **Writes are server-authoritative**: `uploadSketch(image)` and `redeemCode(code)`
  send no userId/pairId — the backend resolves them from the token + pairing record.
- **Pairing is symmetric**: either partner redeeming the other's code binds
  both. `usePairing` polls `GET /pair` so an unpaired partner advances on its own.
- **Config**: see `.env.example` + `README.md`. App boots to a setup notice when
  `VITE_AUTH0_*` are unset rather than crashing.

## shadcn

shadcn is installed and **MCP tools are available** for it in this session. Use MCP to look up, browse, and add components rather than guessing CLI commands.

- Components live in `src/components/ui/` and are generated by shadcn — avoid hand-editing them.
- The project uses **Base UI** primitives (not Radix), which is what this version of shadcn ships.
- To add a new component: use the shadcn MCP tool `get_add_command_for_items`, then run the printed command.
- `cn()` from `@/lib/utils` is the class-merging helper used throughout.

## Icons

Use **react-icons, Ionicons 5 pack only**:

```tsx
import { IoTrash, IoBrush, IoColorPalette } from 'react-icons/io5'
```

Do not import from other react-icons packs (fa, md, hi, etc.). Ionicons 5 covers all common UI needs; if something seems missing, look harder — the pack is large.

**Exception:** the tool selector (`ToolSelector.tsx`) uses the **Phosphor pack** (`react-icons/pi` — `PiPen`, `PiPencil`, `PiHighlighter`, `PiEraser`) because its line-art tool glyphs match the UI better than Ionicons here.

## Styling rules

- Tailwind utility classes for all layout and static styles.
- Inline `style={{}}` only for **runtime-dynamic values** that cannot be expressed as Tailwind classes (e.g. a hex color from state, a computed pixel size).
- Arbitrary Tailwind values (`text-[13px]`, `border-[1.5px]`, `rounded-[10px]`) are fine.
- No CSS modules, no styled-components, no extra CSS files beyond `index.css`.
