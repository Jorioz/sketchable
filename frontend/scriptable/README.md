# Sketchable — Scriptable widget

`sketchable-partner.js` is a [Scriptable](https://scriptable.app) script (iOS)
that shows your partner's most recent sketch on your home screen and refreshes
periodically. It's the same script the Sketchable app hands you during
onboarding — already filled in with your credentials.

## How users get it (the onboarding flow)

In the Sketchable web app's setup checklist:

1. **Add to Home Screen** — install the PWA.
2. **Sign in with Google** — Auth0 login.
3. **Get the Scriptable app** — from the App Store.
4. **Copy your script** — the app calls `POST /me/script-token` to mint a
   long-lived, read-only token, fills it into this template, and copies the
   whole script to the clipboard. Paste it into a new script in Scriptable.

Then long-press the home screen → add a **Scriptable** widget → choose the
script. iOS refreshes it on its own schedule.

## What's in the script

The app fills three constants at the top:

| Constant | Source |
|---|---|
| `API_TOKEN` | `POST /me/script-token` → `token` (read-only, ~1 year) |
| `API_BASE_URL` | the deployed API base URL (`VITE_API_BASE_URL`) |
| `CDN_DOMAIN` | optional; only if the API returns `url: null` for sketches |

This file in the repo is the **template** — those constants are left as
`__PLACEHOLDERS__`. The canonical copy users receive is generated by
`frontend/src/lib/scriptableScript.ts`; keep the two in sync if you edit the
widget logic.

## Security

`API_TOKEN` is a bearer credential. It is **read-only**: the backend rejects any
non-GET route (upload, pairing, username, account deletion, minting more tokens)
for a script token. Worst case if it leaks, someone can view the couple's
sketches — they can't modify anything. Rotate by changing the backend's
`SCRIPT_TOKEN_SECRET` (invalidates all issued tokens) and re-copying the script.

## Notes

- The widget falls back to the last cached sketch when offline.
- Running the script inside the Scriptable app previews the widget.
